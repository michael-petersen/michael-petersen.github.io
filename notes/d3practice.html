D3 Basics

Web-based, interactive visualisations can really help with science storytelling. However, I build the visualisations so infrequently that I regularly forget how I made a visualisation previously. This note consolidates some of the things I repeatedly forget when designing D3 visualisations. D3 stands for data driven documents, and is my preferred javascript library for making online visualisations. Unfortunately, D3 can feel like a bit of black magic, and I find myself repeatedly copying code when I should almost certainly be writing something new.


The key start begins with firing up a local server to execute the D3 javascript code. Really, this step is common for any javascript code that we want to prototype. Ideally this will be run in whichever directory holds your webpage, or you will end up navigating a lot of your computer directory structure.

python -m http.server 8889

Now head to http://localhost:8889/ to see the local webpage in action (note that you might not want to use 8889 if you have a bunch of jupyter servers running: it could already be taken!).


The most useful link for me getting started with visualisations is 'The D3 Graph Gallery'.


To debug: In Chrome, View->Developer->JavaScript Console. The first thing you will probably notice is that you have some errors. The console is really helpful for debugging those! However, you can also use the console to interact with your data.


For a first demo, I have reminded myself how to make simple line plots, with a single additional option to select between diffrent data sets. I've started with MLB data, which is usually an excellent playground!


I have quite a few goals for interactive visualisations, starting with some real-time updates of CSVs that are immediately ingested into a webpage.


Glossary: SVG, structured vector graphics (where we can embed graphics directly in html);
